# @5W1H Framework - The Analysis Tool

## The Framework

**5 W's and 1 H:**
- **WHO** = Who is involved?
- **WHAT** = What needs to be done?
- **WHEN** = When does it need to be done?
- **WHERE** = Where does it need to be done?
- **WHY** = Why does it need to be done?
- **HOW** = How will it be done?

## The Purpose

### Determining the @ASK

**@5W1H determines:**
- **The actual ask** = What is really being requested?
- **The context** = What is the full context?
- **The requirements** = What are all the requirements?
- **The constraints** = What are the constraints?
- **The solution** = What is the solution?

### Comprehensive Analysis

**@5W1H provides:**
- **Complete picture** = Full understanding of the situation
- **All factors** = All relevant factors considered
- **Clear requirements** = Clear understanding of requirements
- **Actionable plan** = Actionable plan based on analysis
- **No assumptions** = No assumptions, only facts

## The Application

### In Cybersecurity

**@5W1H cybersecurity analysis:**
- **WHO** = Who is the threat? Who is affected?
- **WHAT** = What is the threat? What needs to be secured?
- **WHEN** = When did it happen? When does it need to be fixed?
- **WHERE** = Where is the vulnerability? Where is the threat?
- **WHY** = Why is it a threat? Why does it need to be secured?
- **HOW** = How did it happen? How will it be mitigated?

### In Action Planning

**@5W1H action planning:**
- **WHO** = Who will execute? Who is responsible?
- **WHAT** = What needs to be done? What is the task?
- **WHEN** = When will it be done? When is the deadline?
- **WHERE** = Where will it be done? Where are the resources?
- **WHY** = Why is it important? Why does it need to be done?
- **HOW** = How will it be done? How will it be executed?

## The Process

### Step 1: WHO

**Identify stakeholders:**
- **Who is involved?** = All parties involved
- **Who is responsible?** = Who has responsibility?
- **Who is affected?** = Who will be impacted?
- **Who needs to know?** = Who needs to be informed?

### Step 2: WHAT

**Define the task:**
- **What needs to be done?** = The actual task
- **What are the requirements?** = All requirements
- **What are the constraints?** = All constraints
- **What is the deliverable?** = What is the output?

### Step 3: WHEN

**Establish timeline:**
- **When does it start?** = Start time
- **When is it due?** = Deadline
- **When are milestones?** = Key milestones
- **When is it complete?** = Completion time

### Step 4: WHERE

**Identify location:**
- **Where will it be done?** = Physical or digital location
- **Where are resources?** = Where are needed resources?
- **Where is the data?** = Where is relevant data?
- **Where is the output?** = Where will output be delivered?

### Step 5: WHY

**Understand purpose:**
- **Why is it important?** = The importance
- **Why now?** = The urgency
- **Why this way?** = The rationale
- **Why this person?** = The reason for assignment

### Step 6: HOW

**Plan execution:**
- **How will it be done?** = The method
- **How will it be verified?** = Verification method
- **How will it be delivered?** = Delivery method
- **How will success be measured?** = Success metrics
