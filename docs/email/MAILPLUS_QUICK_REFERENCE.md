## 🔐 Authentication Methods

### Gmail
- **Method**: OAuth 2.0 (XOAUTH2)
- **Required**: Google account with OAuth consent
- **No App Passwords**: OAuth only for Workspace accounts

### ProtonMail
- **Method**: Bridge-generated credentials
- **NOT**: Your ProtonMail web password
- **Location**: Check Bridge UI for credentials

### MailPlus
- **Method**: Username/Password
- **Username**: Your MailPlus email address
- **Password**: MailPlus account password

## 🌐 Network Addresses

### Common Patterns
```
localhost: 127.0.0.1

            OR [Host IP] (Linux Docker)
```

2. **Command**: `ipconfig` (Windows) or `ifconfig` (Linux/Mac)
3. **Router**: Check DHCP client list

## ✅ Quick Test Commands

### Test IMAP Connection
```bash
# Test MailPlus IMAP
telnet [NAS IP] 993

# Test Proton Bridge IMAP (from host)
telnet 127.0.0.1 1143
```

### Test SMTP Connection
```bash
# Test MailPlus SMTP
telnet [NAS IP] 587

# Test Proton Bridge SMTP (from host)
telnet 127.0.0.1 1025
```

### Test from N8N Container
```bash
# If N8N in Docker, test Bridge connection
docker exec -it n8n-container telnet host.docker.internal 1143
```

## 🚨 Common Issues & Quick Fixes

### "Cannot connect to server"
- ✅ Check NAS IP address
- ✅ Verify firewall allows ports 993, 587
- ✅ Ensure MailPlus Server is running
- ✅ Test with telnet command

### "Authentication failed"
- ✅ Verify username/password
- ✅ Check account is activated in MailPlus
- ✅ For Proton: Use Bridge credentials, not web password
- ✅ For Gmail: Re-authenticate OAuth

### "SSL/TLS error"
- ✅ Accept self-signed certificate
- ✅ Enable "Allow self-signed certificates"
- ✅ Verify encryption settings match

### "Port already in use" (Bridge)
- ✅ Change Bridge ports in Settings → Advanced
- ✅ Update all configurations with new ports
