# Workflow Stage Mantras - Core Morals/Values/Ethics

## Principle

**All mantras are CORE MORALS/VALUES/ETHICS**

These are not just workflow steps - they represent fundamental principles that guide all work.

## Workflow Stage Mantras

### Development Stage
**Mantra: "DOCUMENT, DOCUMENT, DOCUMENT"**

**Core Value:** Knowledge sharing, transparency, accountability

**Principles:**
- Logging = Documenting
- Code comments and docstrings
- README files and guides
- Knowledge preservation
- Transparency in all work

**Why:** Documentation is a moral obligation. It ensures knowledge sharing, transparency, and accountability. Every line of code, every decision, every process should be documented.

### IT/Testing Stage
**Mantra: "DELEGATE, DELEGATE, DELEGATE"**

**Core Value:** Trust, efficiency, scalability, responsibility

**Principles:**
- Automate repetitive tasks
- System testing and validation
- Task distribution
- Trust in automated processes
- Scalable solutions

**Why:** Delegation is a core value. It represents trust, efficiency, and scalability. Systems should handle what they can, freeing humans for higher-level work.

## Workflow Progression

```
DEVELOPMENT → DOCUMENT, DOCUMENT, DOCUMENT
     ↓
IT/TESTING → DELEGATE, DELEGATE, DELEGATE
```

### The Progression Represents:

1. **Maturity**: From creating to validating
2. **Trust**: From individual to system
3. **Scale**: From manual to automated
4. **Ethics**: From personal to shared responsibility

## Core Morals/Values/Ethics

### Documentation (Development)
- **Moral**: Knowledge should be shared, not hoarded
- **Value**: Transparency enables collaboration
- **Ethic**: Accountability through documentation

### Delegation (IT/Testing)
- **Moral**: Trust enables efficiency
- **Value**: Automation scales human capability
- **Ethic**: Responsibility through proper delegation
