## Character Stats Analysis

### Intent Analysis

**What It Measures:**
- Why does this author write code?
- What are their primary motivations?
- What patterns emerge?

**Metrics:**
- Fix-focused: High intent to fix issues
- Feature-focused: High intent to add features
- Refactor-focused: High intent to improve code
- Optimize-focused: High intent to improve performance

### Consistency Analysis

**What It Measures:**
- How consistent are commit patterns?
- Message length consistency
- Style consistency
- Pattern reliability

**Metrics:**
- Score: 0.0 - 1.0 (higher = more consistent)
- Patterns: Message length, style, frequency
- Analysis: Consistency assessment

### Performance Analysis

**What It Measures:**
- How well does code perform?
- Performance-focused commits
- Optimization patterns
- Bottleneck awareness

**Metrics:**
- Score: 0.0 - 1.0 (higher = better performance)
- Indicators: Optimize, performance, speed keywords
- Analysis: Performance assessment

### Reputation Analysis

**What It Measures:**
- Author's track record
- Positive vs negative contributions
- Reliability history
- Trustworthiness

**Metrics:**
- Score: 0.0 - 1.0 (higher = better reputation)
- History: Positive/negative indicators
- Analysis: Reputation assessment
