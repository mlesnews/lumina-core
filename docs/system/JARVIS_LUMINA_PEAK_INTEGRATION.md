## Integration Flow

### Code Generation Flow

1. **User Request** → IDE (Cursor/VS Code/Abacus)
2. **Kilo Code** → Processes with @Peak patterns (REQUIRED)
3. **JARVIS** → Orchestrates workflow
4. **Lumina** → Manages extensions and content
5. **Watcher Utau** → Researches patterns, generates sparks
6. **JARVIS** → Receives viable sparks
7. **R5** → Aggregates knowledge
8. **Result** → Returned to user

### Pattern Registration Flow

1. **Pattern Detected** → @Peak annotation
2. **Kilo Code** → Extracts pattern
3. **Watcher Utau** → Researches pattern
4. **JARVIS** → Registers pattern
5. **Lumina** → Indexes pattern
6. **R5** → Aggregates pattern knowledge
7. **Pattern Available** → For future use

### Spark Transmission Flow

1. **Watcher Utau** → Generates @sparks from research
2. **Viability Assessment** → Evaluates spark
3. **Azure Vault** → Protects spark as secret (#azvault)
4. **JARVIS** → Receives viable sparks
5. **Lumina** → Indexes spark
6. **R5** → Aggregates spark knowledge
