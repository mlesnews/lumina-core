## 📋 Core Workflows

### Pre-Change (Before Starting Work)
1. Create/Reference Issue
2. Create Feature Branch
3. GitLens Analysis (repository, conflicts, history)
4. Risk Assessment (Decision Tree + R5 + Risk Matrix)
5. Get Pre-Approval (if High/Critical risk)

### Post-Change (After Work Complete)
1. Verification (tests, linting, functionality)
2. Create/Update PR
3. Validation (CI/CD, reviews, feedback)
4. Re-Verification (final checks, approvals)
5. Merge & Cleanup

### Risk-Based Approvals
- **Low**: 1 reviewer, Basic CI/CD
- **Medium**: 2 reviewers, Full CI/CD + Security
- **High**: 3+ reviewers + maintainer, Full CI/CD + Security + Performance + QA
- **Critical**: 4+ reviewers + maintainer + security, Full Audit + Manual QA
