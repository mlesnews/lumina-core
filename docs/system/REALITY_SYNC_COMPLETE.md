## RAID Mirroring Repair Logic

### How It Works

1. **Compare Checksums** (like RAID)
   - Calculate MD5 checksum for each reality
   - Compare checksums
   - Detect mismatches

2. **Determine Severity**
   - Critical: Checksum mismatch
   - High: Data hash mismatch
   - Medium: Timestamp drift
   - Low: Minor differences

3. **Apply Repair Action**
   - **Repair**: Rebuild experiment from control (critical)
   - **Merge**: Merge changes (high)
   - **Resolve**: Resolve conflicts (medium)
   - **Ignore**: Minor differences (low)

4. **Sync Realities**
   - Copy control state to experiment
   - Update checksums
   - Mark as IN SYNC
