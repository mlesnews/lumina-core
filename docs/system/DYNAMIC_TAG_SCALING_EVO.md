## 🔄 Dynamic Scaling Module Logic

### How It Works

Instead of **fixed static rules**, tags now use **dynamic scaling factors**:

1. **System Load** (0.0 - 1.0)
   - High load → Reduce scale (efficiency)
   - Low load → Increase scale (performance)

2. **Urgency** (0.0 - 1.0)
   - High urgency → Increase scale (speed)
   - Low urgency → Normal scale

3. **Resource Availability** (0.0 - 1.0)
   - More resources → Increase scale
   - Fewer resources → Decrease scale

4. **Historical Effectiveness** (0.0 - 1.0)
   - High effectiveness → Maintain/increase scale
   - Low effectiveness → Decrease scale

5. **Tag Combinations**
   - Effective combinations → Boost scale
   - Complex combinations → Slight reduction

### Scale Factor Calculation

```
scale_factor = base_scale 
             × load_factor 
             × urgency_factor 
             × resource_factor 
             × effectiveness_factor 
             × combination_factor
```
