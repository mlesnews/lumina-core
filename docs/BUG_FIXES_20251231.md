# Bug Fixes - December 31, 2025

## ✅ Bug 1: Stop Phrase Substring Matching - FIXED

### Problem
The stop phrase detection used substring matching (`phrase in text_lower`) with common words like "stop", "cancel", and "done". This caused legitimate commands containing these words to be incorrectly treated as stop commands.

**Example Issue:**
- User says: "jarvis, cancel my appointment"
- System matches "cancel" as substring in "cancel my appointment"
- Command is silently lost - system resets to trigger-waiting mode without processing

### Solution
Changed from simple substring matching to regex-based pattern matching with word boundaries:

1. **Exact phrase matching** for stop commands
2. **Word boundary checks** to avoid false positives
3. **Explicit stop phrases** list for unambiguous commands

**Fixed Code:**
```python
# Use word boundaries and regex patterns
stop_phrases = [
    r"^(stop|cancel|done|never mind|that's all)$",  # Exact match
    r"^(stop|cancel|done|never mind|that's all)\s",  # At start
    r"\s(stop|cancel|done|never mind|that's all)$",  # At end
    # Plus explicit unambiguous phrases
]

explicit_stop_phrases = [
    "stop listening",
    "that's all",
    "never mind",
    "cancel that",
    "done talking",
    "finished talking"
]
```

**Location:** `scripts/python/jarvis_always_listening.py` lines 252-264

## ✅ Bug 2: Trigger State Reset Outside on_command Block - FIXED

### Problem
The trigger state reset (`waiting_for_trigger = True`) was nested inside the `if self.on_command:` block. When `on_command` is not set (it's declared as `Optional` and defaults to `None`), the trigger state is never reset after processing a command.

**Impact:**
- After first trigger word detection, system stays in "command mode" indefinitely
- All subsequent speech treated as commands instead of waiting for "jarvis" trigger
- No way to return to trigger-waiting mode without restart

### Solution
Moved trigger state reset logic outside the `if self.on_command:` conditional block so it executes regardless of whether a command handler is registered.

**Fixed Code:**
```python
# Check if it's a command and process
if self.on_command:
    response = self.on_command(text)
    if response:
        self.speak(response)

# After processing, return to waiting for trigger
# IMPORTANT: This must be outside the on_command block so trigger state
# resets even when no command handler is registered
self.waiting_for_trigger = True
self.trigger_detected = False
logger.info(f"✅ Ready - waiting for '{self.trigger_word}' trigger word")
```

**Location:** `scripts/python/jarvis_always_listening.py` lines 291-300
